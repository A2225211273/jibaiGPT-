# 禾序无边框桌面运行态设计

状态：已确认
日期：2026-07-15
产品：禾序 · Codex 用量助手
开发者：沐青禾

## 目标

把当前带 Windows 原生标题栏的 Tauri 窗口改成与正式 UI 稿一致的一体化悬浮卡片，同时保留真实数据读取、窗口拖动、最小化和关闭能力。

正式视觉基准为 [hexu-ui-desktop-v1.png](../../../design/ui/hexu-ui-desktop-v1.png)。本次只修正桌面运行态的窗口外观和交互，不改变 Codex `app-server` 数据协议，也不提前实现窗口吸附、托盘或工作计时。

## 已确认选择

- 采用透明 Tauri 外层窗口与独立 CSS 卡片层。
- 卡片本体保持 `380×520`。
- 卡片四周各保留 `12px` 透明阴影空间，外层窗口为 `404×544`。
- 系统标题栏和系统边框全部关闭。
- 最小化和关闭按钮平时隐藏，鼠标进入顶部区域或键盘焦点进入控件组时淡入。
- 不提供最大化按钮；窗口固定尺寸且不可缩放。

## 窗口结构

```text
404 × 544 透明 Tauri 窗口
┌────────────────────────────────────┐
│ 12px 透明阴影区                    │
│   ┌────────────────────────────┐   │
│   │ 380 × 520 禾序内容卡片     │   │
│   │ 圆角、边框、阴影都由 CSS   │   │
│   └────────────────────────────┘   │
│                    12px            │
└────────────────────────────────────┘
```

Tauri 主窗口使用以下行为：

- `decorations: false`
- `transparent: true`
- `width: 404`
- `height: 544`
- `minWidth: 404`
- `minHeight: 544`
- `resizable: false`
- `fullscreen: false`
- 启动时居中

外层窗口不依赖系统标题栏阴影。卡片使用 CSS 阴影并在透明边距内完整渲染，避免阴影被窗口边界裁切。当前阶段不启用强制置顶，避免无边框窗口持续遮挡用户工作区。

## 卡片视觉

桌面运行态恢复正式稿的卡片属性：

- 固定 `380×520`
- `24px` 圆角
- 雾白/白色面板背景
- 浅灰青细边框
- 柔和的下落阴影
- 顶部品牌、套餐状态、设置按钮和三段序列波保持正式稿信息层级

运行态必须使用独立的高优先级样式作用域，不能再被浏览器设计板的 `max-width: 720px` 响应式规则覆盖。状态序列波、标题字号、用量行比例、汇总区和页脚尺寸均以正式桌面稿为准。

`html`、`body`、`#root` 与运行态外壳都使用透明背景；只有 `desktop-panel` 绘制实体面板。透明四角不能出现白块、黑块或系统标题栏残影。

## 顶部拖拽与窗口控件

运行态新增真正的 `WindowControls`，浏览器设计板继续保留仅用于稿图展示的 `WindowDots`，两者不共用伪交互。

顶部交互规则：

1. 面板头部的非交互空白区域是拖拽区。
2. 品牌区可以参与拖拽；设置、最小化和关闭按钮不能触发拖拽。
3. 窗口控件默认 `opacity: 0` 且不接收鼠标事件。
4. 鼠标进入头部或控件获得键盘焦点时，控件在 `160ms` 内淡入。
5. 鼠标离开且控件失去焦点后淡出。
6. 最小化按钮调用当前 Tauri 窗口的 `minimize()`。
7. 关闭按钮调用当前 Tauri 窗口的 `close()`。
8. 关闭按钮仅在悬停时出现轻微危险色，不使用大面积红色系统按钮。

窗口控件保持小尺寸、低对比度，视觉位置参考正式稿右上角，但不会与“PLUS · 实时”和设置按钮重叠。键盘用户可通过 Tab 聚焦两个按钮，聚焦时必须显示清晰轮廓。

## 数据与状态

本次不修改真实用量数据流：

```text
React 运行态
  -> Tauri invoke: hexu_usage_snapshot
  -> Rust 启动本机 Codex app-server
  -> account/read + account/rateLimits/read + account/usage/read
  -> 隐私安全快照
  -> 桌面卡片渲染
```

加载、未登录、不支持、读取失败、短周期缺失和长周期真实额度等状态继续沿用当前逻辑。无边框改造不能影响 60 秒刷新、手动刷新或隐私边界。

## 错误处理

- 窗口操作失败时记录开发日志，不弹出阻塞式系统对话框。
- 透明窗口若在目标系统不受支持，Debug 验收必须直接暴露问题，不静默恢复成原生标题栏。
- 真实数据读取失败仍在卡片内部显示原有错误状态，不改变窗口结构。
- 控件隐藏不能导致用户无法退出：键盘聚焦会强制显示控件，Windows 任务栏仍可关闭窗口。

## 验收与测试

自动验证：

- `pnpm build`
- `cargo test --manifest-path src-tauri/Cargo.toml`
- `cargo clippy --manifest-path src-tauri/Cargo.toml --all-targets -- -D warnings`
- `pnpm tauri build --debug --no-bundle`

实机验证：

- 启动后没有 Windows 原生标题栏、边框或标题文字。
- 外层窗口为 `404×544`，内部卡片保持 `380×520`。
- 四角透明，阴影完整，无黑边、白边和滚动条。
- 鼠标移入顶部时控件淡入，移出后淡出。
- 顶部空白区可拖动窗口，设置按钮仍能打开设置面板。
- 最小化与关闭操作真实有效。
- 真实 ChatGPT 登录态用量可以正常加载和刷新。
- 浏览器完整稿、桌面稿、贴附稿与会话卡片不发生视觉回归。
- 在 Windows 10 Build 19041+ 与 Windows 11 x64 上检查 100%、125%、150%、200% DPI。

## 非目标

本次不实现：

- Codex 窗口外侧吸附
- 系统托盘
- 自动置顶开关
- 工作时长和休息提醒
- Codex 插件包
- Windows 正式安装器和签名

这些功能保持在后续独立阶段，避免无边框视觉修正与 HUD 行为混在同一提交中。
